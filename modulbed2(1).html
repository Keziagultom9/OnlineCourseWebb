Cara Memakai AWS

When infrastructure becomes virtual, as with cloud computing, the way developers work with infrastructure changes slightly. Instead of physically managing infrastructure, you logically manage it, through the AWS Application Programming Interface (AWS API). When you create, delete, or change any AWS resource, you will use API calls to AWS to do that.

  •        The AWS Management Console
  •        The AWS Command Line Interface (AWS CLI)
  •        IDE and IDE toolkits
  •         AWS Software Development Kits (SDKs)

AWS Console

Saat pertama kali memulai dengan AWS, orang sering memulai dengan AWS Management Console, konsol berbasis web yang Anda masuki melalui browser. Konsol terdiri dari kumpulan luas konsol layanan untuk mengelola sumber daya AWS. Dengan bekerja di konsol, Anda tidak perlu khawatir tentang skrip atau sintaks. Anda juga dapat memilih Wilayah tertentu yang Anda inginkan untuk layanan AWS.

Setelah bekerja di konsol, Anda mungkin ingin beralih dari penerapan layanan AWS secara manual, mungkin karena Anda telah menjadi lebih akrab dengan AWS atau bekerja di lingkungan produksi yang memerlukan tingkat manajemen risiko. Di sinilah AWS Command Line Interface (CLI) masuk.

AWS CLI

AWS CLI adalah alat sumber terbuka yang memungkinkan Anda membuat dan mengonfigurasi layanan AWS menggunakan perintah di shell baris perintah Anda. Anda dapat menjalankan perintah di Linux atau macOS menggunakan program shell umum seperti bash, zsh, dan tcsh, atau di Windows, di prompt perintah Windows atau di PowerShell. Salah satu opsi untuk memulai dan menjalankan dengan cepat dengan AWS CLI adalah AWS CloudShell, shell berbasis browser yang menyediakan akses baris perintah ke sumber daya AWS. CloudShell sudah diautentikasi sebelumnya dengan kredensial konsol Anda. Alat pengembangan dan operasi umum sudah diinstal sebelumnya, jadi tidak diperlukan instalasi atau konfigurasi lokal.

Dengan pindah ke AWS CLI, Anda dapat membuat skrip atau memprogram panggilan API. Alih-alih menggunakan GUI, Anda membuat perintah menggunakan sintaks AWS yang ditentukan. Salah satu manfaat CLI adalah Anda dapat membuat satu perintah untuk membuat beberapa sumber daya AWS, yang dapat membantu mengurangi kemungkinan kesalahan manusia saat memilih dan mengonfigurasi sumber daya. Dengan CLI, Anda perlu mempelajari sintaks yang tepat untuk membentuk perintah, tetapi saat Anda membuat skrip perintah ini, Anda membuatnya dapat diulang. Ini akan menghemat waktu Anda dalam jangka panjang.
